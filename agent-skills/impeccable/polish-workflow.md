# Impeccable Polish Workflow
Source: https://github.com/pbakaus/impeccable

## Polish Checklist
- **Typography**: Fix pairing, modular scales, and ensure hierarchy is immediately obvious. Use OpenType features where supported.
- **Color & Contrast**: Use tinted neutrals (avoid pure #000 or pure gray). Ensure WCAG AA contrast compliance.
- **Spatial Design**: Use consistent spacing systems and grids. Avoid random padding/margin values.
- **Interaction Design**: Ensure clear loading patterns, visible focus states, and logical form layouts.
- **UX Writing**: Button labels must be descriptive and active. Error messages must be helpful.
- **Responsive Design**: Prioritize mobile-first and fluid layouts. Test specifically for smaller Android widths.

## Anti-Patterns to Avoid
- Don't use overused fonts (default Inter for everything).
- Don't use gray text on colored backgrounds.
- Don't wrap everything in identical boxes/cards.
- Don't use bounce/elastic easing.
- Don't nest cards inside cards.

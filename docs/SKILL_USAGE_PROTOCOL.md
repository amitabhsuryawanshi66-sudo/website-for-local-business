# Skill Usage Protocol

To maintain the "Pro Max" quality of this V2 template, every build or modification task must follow this exact workflow.

## 1. Context Initialization
- **Action:** Read `AGENTS.md` and `agent-skills/` folders.
- **Goal:** Shift state from "Generic LLM" to "Design Engineer."

## 2. Component Design (The 21st.dev Loop)
For any new component or major section:
- **Phase A: Variation Generation.** Internally conceptualize (A) Minimal Luxury, (B) Bold Brand, and (C) Editorial Boutique.
- **Phase B: Selection.** Choose the style that best fits the Pune local business profile (usually Editorial Boutique).
- **Phase C: Implementation.** Code the selected variant using Pro Max tokens.

## 3. The "Impeccable" Polish Pass
Before finalizing code:
- **Typography:** Tighten headings, ensure body font is ≥18px on mobile.
- **Contrast:** Check button contrast against brand accents.
- **Spatial:** Ensure `container-px` and `section-py` are used correctly.

## 4. Motion Engineering (Emil Kowalski)
- **Reveals:** Apply `[0.16, 1, 0.3, 1]` easing to all `V2Section` reveals.
- **Feedback:** Ensure all buttons have a spring-based `whileTap` scale effect.
- **Sequencing:** Use `staggerChildren` to prevent the UI from appearing all at once.

## 5. Mobile-First (Android) QA
- **Step 1:** Audit at 360px and 390px widths.
- **Step 2:** Ensure the "Sticky WhatsApp" button doesn't overlap package text.
- **Step 3:** Verify that all tap targets are accessible to the thumb in the lower 60% of the screen.

## 6. Deployment Verification
- **Action:** Run `npm run build`.
- **Proof:** Report the bundle size and confirm no console errors in the output log.
